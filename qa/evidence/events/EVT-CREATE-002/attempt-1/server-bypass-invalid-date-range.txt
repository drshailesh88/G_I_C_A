# Server-Side Bypass Test: Invalid Date Range (End Before Start)
# Packet: EVT-CREATE-002
# Date: 2026-04-13

## Method
Removed HTML `required` attributes, set valid name ("Bypass Test Event") and venue ("Test Venue"),
set startDate to "2026-12-15" and endDate to "2026-12-10" (end before start), then submitted via
button click, bypassing any client-side date validation.

## JavaScript Used (in browser eval)
```js
// Set values via native input value setter to bypass React controlled input
nativeSet.call(nameInput, 'Bypass Test Event');
nativeSet.call(venueInput, 'Test Venue');
nativeSet.call(startDate, '2026-12-15');
nativeSet.call(endDate, '2026-12-10');
// Click submit to trigger server action
```

## Server Response
HTTP Status: 500 (Next.js wraps server action ZodError as 500)
Error displayed in UI:

[
  {
    "code": "custom",
    "message": "End date must be on or after start date",
    "path": ["endDate"]
  }
]

## Validation Source
The date range check is a Zod `.refine()` on the createEventSchema at
src/lib/validations/event.ts:45-48:
```ts
.refine(
  (data) => new Date(data.endDate) >= new Date(data.startDate),
  { message: 'End date must be on or after start date', path: ['endDate'] },
);
```

## Result
- Server-side Zod validation REJECTED the invalid date range
- Clear error message: "End date must be on or after start date"
- Page remained on /events/new (no redirect)
- No event record created with invalid dates
- Rejection happens at Zod validation (line 68 of event.ts: createEventSchema.parse(raw))
  BEFORE any database insert operation (line 72-91)
