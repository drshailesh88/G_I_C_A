# Session 1: Whova — Complete UX Teardown
**Modules:** Event Management (#3), Registration (#4), QR & Attendance (#11), Dashboard (#13)
**URL:** https://whova.com
**Time:** 2-3 hours

## 1A — Event Creation & Management
- Document complete event creation flow step by step
- Every field: name, type (text/dropdown/date/toggle/upload), required vs optional
- Save behavior: confirmation, redirect, toast?
- ON/OFF toggles for optional features
- Multi-session/track creation within an event
- Edit after creation — versioning?
- Publish vs Draft concept
- Event list/overview — past vs upcoming display

## 1B — Agenda / Schedule Management
- How to add sessions to an agenda
- Multi-track/parallel halls
- Per-session fields: title, time, duration, room, speaker, description
- Speaker/moderator assignment to sessions
- Drag-and-drop rearrangement
- Organizer vs attendee view of agenda
- Schedule change notifications
- PDF export
- Multi-day navigation
- Layout type: grid/calendar/list/cards? Filters?

## 1C — Registration Flow
### Organizer Side
- Registration form setup
- Default vs custom fields (types: text, dropdown, checkbox, file upload)
- Attendee list management and export format
- Approval workflow vs automatic
- Registration metrics display

### Attendee Side
- Public registration page design
- Step-by-step signup flow
- Confirmation (email/in-app/registration number)
- Post-submission editing

## 1D — Dashboard & Reporting
- Home dashboard layout and metrics
- Charts and visual indicators
- Every report type available
- Export formats (CSV, PDF, Excel)
- Per-event archive access
- Sidebar navigation — every menu item
- Dashboard layout: top section, sidebar, main content

## 1E — QR Check-in
- QR code generation per attendee
- Scanner interface (web vs native, phone/iPad)
- Information shown on scan
- Check-in analytics (attendance counts, hall popularity)
- Checked-in vs not-checked-in views
