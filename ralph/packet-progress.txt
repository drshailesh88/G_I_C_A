## Patterns

- DB mock for queries ending in `.where()` (no `.limit()`): add `.then()` and `.catch()` to the chain object so it is thenable. Registration tests use `mockReturnThis()` but that only works when the query ends in `.limit()` or `.orderBy()`.
- `withEventScope` should be mocked in unit tests (`vi.mock('@/lib/db/with-event-scope', ...)`) so Drizzle column references are not needed in tests.
- Upsert pattern: select existing → if found update, else insert. Test the two branches separately with `chainedSelect([])` (no record) vs `chainedSelect([{id:'...'}])`.
