## Codebase Patterns

<!-- Reusable patterns discovered during builds. Ralph writes here. -->

- issueCertificate retry loop must use db.transaction() — tests mock mockDb.transaction and count calls
- Supersession updates must be a single .set() call with status, supersededById, and updatedAt together (after insert, not before)
- Person/template validation stays OUTSIDE the transaction retry loop; only the cert-number-dependent logic goes inside
- Transaction-level retry uses PG error class 40 (code.startsWith('40')) to avoid conflicting with class-23 collision/constraint tests
- Stryker baseline with locked stryker.config.json (no ignoreStatic) cannot complete on laptop hardware — static mutants (26% of total, ~2612 mutants) dominate runtime and exceed the 60-min controller timeout. Shadow config files are LOCKED; partial baseline (72/102 scored modules) is the acceptable outcome per the PRD's outcome ladder.
- Controller's parseStrykerReport uses (killed+survived+timeout) as denominator — files with only NoCoverage or Ignored mutants get score=null and are skipped by applyMutationMeasurement
- API route pattern: type Params = Promise<{ eventId: string }>; await params to destructure; paramsSchema validates UUID; assertEventAccess for auth; safeParse body with Zod; catch errors and map to status codes per contract
- Certificate endpoints require super_admin or event_coordinator (not ops) — need extra role check beyond assertEventAccess's WRITE_ROLES which includes ops
- API request bodies use snake_case per contract; Zod .transform() maps to camelCase for internal action functions
- Bulk endpoints with Redis locking: use SET NX EX with JSON metadata (lock_holder, started_at); store lockKey in Inngest payload so handler can DEL on completion; mock @upstash/redis constructor in tests
- Bulk generation uses per-cert atomicity: each recipient gets its own try/catch + db.transaction(); failures reported in errors array, not thrown; no intermediate/pending status rows
- CME validation uses createCmeVariablesSchema(maxHours) factory — event duration fetched in route handler, schema created at request time with runtime max; non-CME types skip this validation

- notification_log upsert: upsertLogEntry uses INSERT...ON CONFLICT(idempotency_key) DO UPDATE with sql`${notificationLog.attempts} + 1` for atomic increment. Never creates a second row for same key. Status/error/timestamps updated on conflict.

- Cascade debounce: debouncedEmitCascadeEvent uses two Redis keys per (eventId, sourceEntityType, sourceEntityId): debounce lock key with SET NX EX 5 and buffer key for merged changeSummary. First call in window emits via emitCascadeEvent with merged payload + _bufferKey; subsequent calls merge into buffer and skip emit. Different source entities use distinct keys so never coalesce.

- issueCertificate select sequence order: event status → person exists → event_people attachment → template active → (inside tx) existing certs + cert numbers. All tests using chainedSelectSequence must include the event_people row after person row.
- assertEventAccess throws EventNotFoundError (→ 404) for unmembered cross-event access; route handlers use instanceof check. Test mocks must export EventNotFoundError via vi.hoisted() to avoid "Cannot access before initialization" errors.
- assertEventAccess throws ForbiddenError (→ 403, statusCode=403) for role-denial within an assigned event (e.g. read_only + requireWrite). Route handlers catch non-EventNotFoundError as 403. Tests mocking assertEventAccess for write-denial should throw new Error('forbidden') or ForbiddenError.
- eventId mismatch guard: import assertEventIdMatch from src/lib/auth/event-id-mismatch.ts. Call after body parse with { urlEventId, bodyEventId: body.eventId, userId, endpoint }. Catches EventIdMismatchError → 400 { error: 'eventId mismatch' }. Sentry event emitted with tags { module: 'tenancy', kind: 'eventId-mismatch' } and extra { urlEventId, bodyEventId, endpoint }.
- Cross-event 404 sanitizer: import crossEvent404Response from src/lib/auth/sanitize-cross-event-404.ts. Use instead of NextResponse.json(null, { status: 404 }) in EventNotFoundError catch blocks. Returns { error: 'Not Found' } with no event-identifying info. sanitize404Body(body) available for custom sanitization.

- Read-only disabled buttons: import useRole() in client component, use canWrite to conditionally render Link (writable) vs disabled button with aria-disabled="true" + disabled attribute + opacity-50 + cursor-not-allowed. Cancel buttons add !canWrite to existing disabled condition. Pattern applies to any module needing read_only UI gating.

- Variable snapshot: resolveVariablesSnapshot() in src/lib/cascade/variables-snapshot.ts resolves person contact + domain vars at emit time and returns Object.freeze(snapshot). Handlers check payload.variables: if present, use directly (no DB re-query); if absent, fall back to resolvePersonContact per-notification. This preserves existing test mock ordering (vi.clearAllMocks doesn't clear mockReturnValueOnce queue, so DB call count must stay consistent).
- GET person-detail pattern: check event_people attachment first (404 if not found), then fetch person master data, then event-scoped attributes (travel, sessions, registration) all filtered by eventId+personId. Response uses snake_case for person fields. No requireWrite needed for read endpoints.

- Inngest function wiring: 7 cascade events mapped to 7 independent Inngest functions (cascade-{event} id pattern). Each has retries:3 and triggers on its respective conference/* event. New handlers (registration, session, certificate) follow the same catch-and-log-never-throw notification pattern as travel/accommodation handlers.
