# Ralph Manual Audit Progress

## Audit Patterns

<!-- Reusable risks, attack ideas, and verification patterns go here. -->
- Desktop/tablet navigation must enforce the same RBAC as mobile menus and server entrypoints; when a page is super-admin-only, hide its nav link and lock the backing server actions, not just the client UI.
- For cascade handlers that accept person IDs from snapshots or payloads, add a defense-in-depth join through `event_people` before notifying; do not trust version metadata or payload person arrays as sufficient event isolation.
- Public success or confirmation routes must resolve the event context from the URL and scope any opaque ID lookup with that event; never trust a bare `registrationId` or similar UUID to imply the correct event.
- Global people mutations that repoint event-owned rows must derive the touched `eventId`s from every dependent table and reject non-super-admin writes outside the caller's active assignment set.
